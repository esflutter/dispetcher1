## Notes
- **Базовая ширина фрейма — 375 × 812** (iPhone X, iOS safe-area). Все экраны фиксированы на эту ширину; «длинные» экраны увеличены только по высоте (формы карточки исполнителя 1888, фильтры каталога 2308, создание услуги 1871–2110, мой график 1492). Для Flutter — `MediaQuery.size.width` базируется на 375 dp, скейл по ширине.
- Сетка/паддинги: повторяющиеся горизонтальные отступы 16 px и 20 px, контейнеры списков выровнены по 16; для Flutter использовать `EdgeInsets.symmetric(horizontal: 16)` как базу.
- Темы: дизайн только **light**, dark-режим не сделан.
- Платформа: только **iOS** (статус-бар «9:41», SF Pro в часах, indicators iOS). Для Android всё-равно подойдёт — никаких iOS-only метафор в контенте нет.
- Файл **не использует Figma Variables / Styles** — нет ни одного опубликованного цвета или текст-стиля. Все токены придётся завести в Flutter вручную (см. палитру и таблицу типографики выше).
- Бренд-цвет — **оранжевый `#FFAC26`** с tint-фоном `#FFF9F0`. Зелёный `#05BD0B` — единственный success-цвет (статусы заказов).
- Иконки нижней навигации в макете подписаны: «Каталог», «Заказы», «Профиль», «Поддержка» — итого 4 таба; «Подписка»/«Карточка исполнителя» — внутренние страницы из «Профиля».
- В разделе «Заказы» есть две Notification (343×66) — это макеты iOS push-уведомлений, не экраны приложения.
- На странице «Концепты» (`61:15552`) на depth=1 children пусто — либо страница пуста, либо контент скрыт от depth-1; для текущей задачи (исполнитель) — нерелевантна.
